# API Documentation Handover

The live OpenAPI schema is generated by drf-spectacular.

Local URLs:

- Swagger UI: `http://localhost:8000/api/docs/`
- Schema: `http://localhost:8000/api/schema/`
- Health: `http://localhost:8000/api/v1/health/`

## Authentication

Use JWT authentication:

- `POST /api/v1/auth/token/`
- `POST /api/v1/auth/token/refresh/`
- `GET /api/v1/auth/me/`

All protected endpoints require:

```http
Authorization: Bearer <access_token>
```

## Tenant Headers

For tenant subdomain or separate-database setups, the backend can derive tenant context from host or `X-Campus-Code`. Authorization is still based on the logged-in user; a user cannot gain access by changing headers.

## Pagination

List endpoints remain backward compatible and return arrays unless pagination is requested.

Use:

```text
?page=1&page_size=50
```

or:

```text
?paginate=true
```

## File Downloads

Downloads must go through protected API routes. Direct public file access should not be used for production student/staff files.

## Payment Webhooks

Webhook handling validates:

- school/campus context
- order ID
- payment ID
- signature
- amount
- gateway/provider

Wrong-school webhook/payment attempts are blocked.
